This directory contains files that explain to others (and possibly your future 
self) your goals and ways of working.

Examples of files to include:
* Assumptions log - Supports QA and enables a reproducible environment
* Data log - Helps manage project data in order to enable a reproducible 
environment
* Project specification - Lays out the purpose of the project and the criteria 
on which it will be judged, which is necessary to validate the project

It may be preferable to store some or all of these documentation files elsewhere, 
like erdm or sharepoint.

## What to do with this README file

If documentation is stored in this directory, this README file can be deleted.

If some or all documents are stored elsewhere, this README file or the README in
the project directory can be used to hold links to the documents (if the main
README file is used and there are no documentation files in this directory, the
whole directory can be deleted).

## Version control

Git version control of data held in Word or Excel formats isn't ideal. 
It's difficult to track changes, and it takes up a lot of space. Therefore, it's
generally preferable to store any such documentation in an alternative version
control platform such as OneDrive/Sharepoint or eRDM.
